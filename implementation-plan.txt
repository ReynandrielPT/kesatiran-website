
Implementation Plan (Updated)
Last updated: 2025-09-25

Purpose of this doc
Give a concise, implementation-ready map of how the site is structured NOW (post casual / friend‑circle redesign) plus what is intentionally deferred. This replaces the earlier corporate/portfolio oriented plan. Anything not listed here is either intentionally removed or out of scope.

--------------------------------------------------------------------------------
1. Core Vision & Tone
--------------------------------------------------------------------------------
We are a small friend circle making things together. The site should feel:
- warm, informal, lowercase, slightly playful
- light, breathable spacing, low cognitive load
- no corporate résumé language (no career highlights, clients served, testimonials, hire us CTA)
- focuses on: who we are, what we’re tinkering with, creative artifacts (songs / images / motion)

Eliminated concepts from old plan: career timeline, testimonials, pricing, availability status blocks, download CV, formal stats KPIs.

--------------------------------------------------------------------------------
2. Tech Stack Snapshot
--------------------------------------------------------------------------------
- Framework: Next.js 15 (App Router)
- React 19 (using `use(params)` pattern for dynamic profile routes)
- Styling: Tailwind + CSS custom properties (design tokens in `globals.css`)
- Data: static JSON under `src/data/*` (no backend). Media under `public/media/*`.
- Components: custom lightweight UI primitives (`Button`, `Card`, `Badge`, `Section`, `Marquee` (currently optional / might be reintroduced), plus contextual pieces like `Navbar`, `Footer` (pending), `MemberCard` (legacy style, pending refresh).
- State: simple React state + a `ThemeContext` for theme family switching.

--------------------------------------------------------------------------------
3. Works Taxonomy (Updated)
--------------------------------------------------------------------------------
Underlying data still uses `type` values: `audio`, `image`, `visual`.
UI labels now map as:
  audio  -> "songs"   (loops, ambient, lo‑fi, anything sonic)
  image  -> "gallery" (photography, posters, design frames, 3d stills)
  visual -> "video"   (motion pieces, animation tests, narrated demos)

Rationale: keep the existing data structure (no migration needed) but surface friendlier, art-centric labels. Future (optional) sub‑filters could derive from tags (e.g., tag: photography, render, poster).

5. Design Tokens (Live Baseline)
--------------------------------------------------------------------------------
Spacing scale (rem based where possible): 0.25,0.5,0.75,1,1.5,2,3,4 (maps to 4px–64px)
Radius: xs 4px, sm 6px, md 10px, lg 16px, xl 24px, pill = full
Shadows: `--shadow-card`, `--shadow-float` (soft layered, no harsh drop)
Color roles (abstract): background, foreground, muted, border, card, accent, accent-foreground, danger, ring.
Motion prefs: default transitions 160–220ms ease-out; respect `prefers-reduced-motion` (fade/translate disabled or simplified).
Focus: 2px outline + 2px offset ring using accent (no inner glow).

11. Contribution Guidelines (Lightweight)
--------------------------------------------------------------------------------
Adding a work:
  - Place media under `public/media/works/...`
  - Add entry to `works.json` (ensure unique id; keep type stable; add tags for potential future filtering)

Adding a song (audio):
  - Prefer compressed (mp3/ogg/opus). Large base64 inline data should be replaced with actual file when available.

Adding a member:
  - Add avatar image in `public/media/members/` (optimize ~ <200KB)
  - Update `members.json` (keep only friendly tags, no corporate roles)

Styling rules:
  - Keep text lowercase except proper nouns.
  - Avoid introducing heavy glassmorphism layering beyond subtle blur already present.
  - Use design tokens; never hard-coded theme hex inside JSX.

Testing passes:
  - Check tab order manually.
  - Toggle themes for contrast regression.
  - Resize: mobile narrow to large desktop.

--------------------------------------------------------------------------------
12. Future Enhancements Outline (Acceptance Hints)
--------------------------------------------------------------------------------
Footer: gradient hairline top border, flex wrap of social pills, 1-line vibe sentence.
Lightbox: add keyboard nav & aria roles; trap focus inside; ESC closes.
Audio queue: simple array of current filtered songs; next/prev wires to skip buttons.
Reduced motion: disable collage animation & marquee (if reintroduced) when user prefers reduced motion.

--------------------------------------------------------------------------------
13. Implementation Notes / Pitfalls
--------------------------------------------------------------------------------
Route params: using React 19 `use(params)` — avoid mixing with older `useRouter().query` patterns.
Avoid giant base64 audio embed in production; replace with file for performance (current sample tracks are placeholder).
When refactoring MemberCard, ensure no hydration mismatch by keeping conditional logic client-safe.
If introducing dynamic imports (e.g., video modal), wrap in `dynamic(..., { ssr: false })` only if needed; prefer SSR-safe first.

--------------------------------------------------------------------------------
14. Minimal Success Criteria (Now)
--------------------------------------------------------------------------------
1. Pages load without TypeScript errors.
2. Works tabs switch instantly; audio play/pause reliable; no uncaught promise rejection on autoplay attempt.
3. Lightbox opens and closes; no scroll bleed (future: add body scroll lock).
4. Focus outline visible on all interactive elements.
5. No lingering corporate copy (CV, hire, clients, etc.).

OVERVIEW
- Purpose: Single-site portfolio for a personal "team circle" (members, profiles, works, audio, images, video, game list).
- Themes: 3 theme families
  1. Formal (light & dark)
  2. Aesthetic / Cafe (light & dark)
  3. Gaming (purple/black) — includes animated RGB strip decoration
- Build: Frontend-only. Static content via JSON/MDX, media in /public/media.
- Deliverable: Clear styling spec + page feature list so it can be used as a prompt for designers/developers.

GLOBAL STYLING TOKENS
- Spacing scale: 4,8,12,16,24,32,48,64 (use rem or px consistent across site)
- Border radius: small 6px, medium 12px, large 24px
- Elevation (shadows): subtle card shadow, medium modal shadow, pronounced floating element shadow
- Breakpoints: mobile <640px, tablet 641–1024px, desktop >1024px
- Typography:
  - Primary: modern sans-serif (e.g., Inter / Poppins) for UI
  - Accent / headers: optional serif for Aesthetic theme (e.g., Merriweather)
  - Scale: h1 36–48px, h2 28–34px, h3 20–24px, body 16px, small 14px
- Motion: short easing (200–350ms) for micro-interactions, 600ms for larger transitions (carousel, modals)
- Accessibility: min contrast ratio 4.5:1 for body text, focus ring 3px visible color

THEME COLOR TOKENS (variables)
- Formal:
  - light: --bg: #FFFFFF, --card: #F7F8FA, --text: #0F1724, --accent: #0B5FFF
  - dark: --bg: #0B1220, --card: #0F1724, --text: #E6EEF8, --accent: #4F8DFE
- Aesthetic (Cafe):
  - light: --bg: #FFF8F2, --card: #FFF1E6, --text: #3E2F2F, --accent: #B87333
  - dark: --bg: #221913, --card: #2B1F1A, --text: #F6EDE7, --accent: #D9A066
- Gaming:
  - base: --bg: #050507, --card: #0D0B10, --text: #EDEBFF, --accent: #7C3AED
  - RGB strip uses animated gradient across neon colors (cyan, magenta, green, purple)
- Notes: All theme colors exposed as CSS variables, used by Tailwind custom properties.

GLOBAL COMPONENTS — STYLE GUIDELINES
1. Navbar
   - Layout: left logo, center nav links (or left if mobile), right theme selector + hamburger
   - Height: 64px desktop, 56px tablet, sticky top
   - Behavior: backdrop blur and slight shadow on scroll; active link underline accent color
   - Mobile: slide-in panel with full-height menu

2. Footer
   - Compact, two columns on desktop (links + contact), stacked on mobile
   - Social icons circular buttons, subtle hover scale

3. Cards (Member / Work / Video)
   - Padding: 16–20px; border-radius medium; hover: lift + soft shadow and slight translateY(-6px)
   - Image: aspect ratio 4:3 or 16:9; object-fit cover; rounded corners matching card

4. Buttons
   - Primary: filled accent with subtle shadow, rounded-pill or medium radius
   - Secondary: outline with border color = muted accent; ghost button for text-only actions
   - Disabled: 50% opacity, cursor not-allowed

4. Avatars / Profile photos
   - Circular 96px desktop, 72px tablet, 56px mobile; border ring matching theme accent (2–3px)

5. Forms / Inputs
   - Rounded inputs, 48px height, label above or floating, clear focus ring (3px)
   - Validation: subtle red text and red border on error

INTERACTIONS & MOTION
- Hover: 1.03 scale, translateY(-3px), shadow increase
- Focus: visible ring using theme accent or neutral high-contrast color
- Entrances: fade + upward slide for sections; lazy-load media with fade-in
- Carousels: snap-to-center, drag-to-swipe on touch, left/right arrow controls, small page indicator

AUDIO PLAYER — STYLING & UX

- Expanded view: large album art, queue list, extra controls (shuffle/repeat)
- Keyboard support: Space toggles play/pause; left/right seek 5–10s
- Persist playback state in localStorage (track index + time)

CAROUSELS & GALLERIES (DETAILS)

- Accessibility: aria-live for track change announcements; focus trap in modal

LIGHTBOX / MODAL
- Fullscreen overlay with center content, ESC to close, click outside to close
- Animations: fade-in overlay + scale-in content
- Lazy-load media only when modal opens

FORMS / CONTACT (if plain frontend)

ACCESSIBILITY MUST-HAVES
